# VPN Director Web UI — Design Spec

## Overview

Web-интерфейс для VPN Director for Asuswrt-Merlin. Standalone Go HTTPS server с Vue 3 SPA, обеспечивающий полный паритет функций с Telegram-ботом. Не зависит от прошивочного httpd — полностью автономное решение на Entware.

## Architecture

### Standalone Go HTTPS Server

Go HTTP server слушает на `0.0.0.0:<port>` (по умолчанию 8444), отдаёт Vue SPA через `go:embed`, предоставляет REST API. Self-signed TLS сертификат генерируется установщиком.

```
Browser / Mobile App
    │
    ├── GET  https://router:8444/          → Vue SPA (embedded in Go binary)
    ├── POST https://router:8444/api/login → JWT token
    └── GET  https://router:8444/api/*     → JSON responses
                │
                ▼
        Go HTTP Server (cmd/webui)
                │
                ▼
        internal/service/* (shared with Telegram bot)
                │
                ▼
        vpn-director.sh / vpn-director.json / shell libs
```

Нет file polling, нет custom_settings, нет service-event — стандартный HTTP request-response.

**Concurrency:** Telegram bot and Web UI both read/write `vpn-director.json`. File locking via `flock()` on a lockfile (`/tmp/vpn-director.lock`) protects against concurrent writes. Implemented in shared `service.ConfigService` (both binaries already use it).

### Why Not Merlin Addon

Merlin Addons API (file polling через httpd, custom_settings с лимитом 8KB, service-event dispatch) добавляет сложность без пользы. Standalone server даёт:

- Нормальный REST API с мгновенным откликом (~50ms vs 1-3s polling)
- Нет лимитов на размер payload
- Возможность WAN-доступа на своём порту
- Будущая совместимость с мобильным приложением (тот же API)
- Zero зависимость от прошивки (всё в `/opt/`)

## Authentication

### JWT + /etc/shadow

**Rate limiting:** In-memory counter per IP address. Max 5 failed login attempts per minute, then 30 second lockout. Prevents brute-force on resource-constrained router.

1. `POST /api/login` — принимает `{"username": "admin", "password": "..."}` 
2. Go-сервер проверяет пароль против `/etc/shadow` через SHA-256 crypt (`$5$` hash)
3. При успехе возвращает JWT token (via `golang-jwt/jwt` library):
   - Подписан HMAC-SHA256 с секретом из конфига (генерируется при установке)
   - Expiration: 24 часа
   - Payload: `{"sub": "admin", "iat": ..., "exp": ...}`
4. Клиент получает JWT двумя способами:
   - **Браузер**: HttpOnly Secure cookie (автоматическая отправка)
   - **Mobile/API**: JSON response body → `Authorization: Bearer <token>`
5. Все `/api/*` эндпоинты кроме `/api/login` требуют валидный JWT
6. `POST /api/logout` — удаляет cookie (JWT stateless, серверной инвалидации нет)

### /etc/shadow Verification

```
/etc/shadow содержит SHA-256 hash ($5$salt$hash) пароля admin.
Go использует crypt() для сравнения: crypt(input_password, stored_salt) == stored_hash.
Универсально работает на всех моделях роутеров.
```

## HTTPS

- Self-signed сертификат генерируется установщиком (`install.sh`) при первом запуске
- Хранится в `/opt/vpn-director/certs/`
- Пути настраиваются в `vpn-director.json` (`webui.cert_file`, `webui.key_file`)
- Пользователь может заменить на свой сертификат (Let's Encrypt и т.д.)
- Go-сервер просто читает файлы при старте

## Frontend: Vue SPA

### Tech Stack

- Vue 3.5 + TypeScript
- Vite (build → single `app.js` bundle)
- No CSS framework — custom dark theme
- No Vue Router — tab-based navigation within single page
- No Vuex/Pinia — lightweight reactive state

### Build

```bash
cd web && npm run build    # → web/dist/index.html + app.js
```

`web/dist/` copied into `server/cmd/webui/web/dist/` at build time, then embedded into Go binary via `go:embed`. Single binary serves everything. SPA fallback: any non-`/api/` request that doesn't match a static file returns `index.html`.

### UI Structure

Single page with 6 tabs:

| Tab | Content | Bot equivalent |
|-----|---------|----------------|
| **Status** | Dashboard: Xray status card, Tunnel Director card, IPSets list, action buttons (Apply, Restart, Stop, Update IPsets), external IP | `/status`, `/restart`, `/stop`, `/ip` |
| **Servers** | Server list cards, select active, import subscription URL | `/servers`, `/xray`, `/import` |
| **Clients** | LAN devices table (IP, hostname, route), add/edit/remove, pause/resume | `/clients` |
| **Exclusions** | Country sets (checkboxes) + IP exclusions (add/remove CIDRs) | `/configure`, `/exclude` |
| **Logs** | Log viewer: filter by source (vpn/xray/bot), configurable line count | `/logs` |
| **Settings** | Version, self-update, raw config JSON view, cert info, port | `/version`, `/update` |

Dark theme consistent with Asuswrt-Merlin admin UI aesthetic:

`/opt/etc/init.d/S98vpn-director-webui`:

```bash
#!/opt/bin/bash
PROCS=webui
ARGS="--config /opt/vpn-director/vpn-director.json"
PREARGS=""
DESC="VPN Director Web UI"
PATH=/opt/sbin:/opt/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

. /opt/etc/init.d/rc.func
```

## Installer Changes

`install.sh` additions:

1. Download `webui` binary (arm64/arm, same pattern as telegram-bot)
2. Generate self-signed TLS certificate if `/opt/vpn-director/certs/` doesn't exist:
   ```bash
   openssl req -x509 -newkey rsa:2048 -keyout server.key -out server.crt \
     -days 3650 -nodes -subj "/CN=vpn-director"
   ```
3. Generate JWT secret if not in config:
   ```bash
   head -c 32 /dev/urandom | base64
   ```
4. Create init script `/opt/etc/init.d/S98vpn-director-webui`
5. Add `webui` section to `vpn-director.json` if missing

## Dev Workflow

### Frontend

```bash
cd web
npm install
npm run dev          # Vite dev server with HMR (localhost:5173)
                     # proxy /api/* → https://router:8444
```

### Backend

```bash
cd server
go run ./cmd/webui --config ../router/opt/vpn-director/vpn-director.json
```

### Build for Router

```bash
cd web && npm run build                              # → web/dist/
cd server && GOOS=linux GOARCH=arm64 go build -o webui ./cmd/webui
scp webui admin@router:/opt/vpn-director/webui       # deploy
```

### CI/CD

GitHub Actions (extend existing workflow):
1. `npm install && npm run build` (Vue SPA)
2. Copy `web/dist/` into `server/` for `go:embed`
3. `GOOS=linux GOARCH=arm64 go build` + `GOARCH=arm`
4. Attach `webui-arm64` and `webui-arm` to GitHub Release
